# enterprise-hth-pages

## 项目简介

`enterprise-hth-pages` 是一个用于归档和发布多个独立 HTML 页面的静态仓库。本仓库不针对任何特定域名或网站，仅作为通用页面存放与展示的载体，适用于企业内部文档、演示页、临时页面等场景的集中管理。

## 目录结构

```
enterprise-hth-pages/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面依赖的静态资源（CSS、JS、图片等）
├── index.html      # 仓库主页索引（可选）
└── README.md       # 本文件
```

- `pages/` 目录下每个子文件夹或文件代表一个独立的页面单元。
- `assets/` 用于存放页面通用的样式、脚本或媒体文件，避免重复引用。
- 根目录下的 `index.html` 可作为页面导航入口，非必需。

## 页面归档说明

- 所有页面均为独立 HTML 文件，可直接通过浏览器打开，无需后端支持。
- 页面之间无强关联，各自保持完整的功能与样式。
- 归档内容遵循通用命名规范，建议使用 `page-xxx.html` 或 `xxx/index.html` 的形式。
- 如需新增页面，请将文件放入 `pages/` 目录，并确保内部资源路径正确。

## 维护说明

- 本仓库由维护者定期更新，新增或修改页面时请保持目录结构清晰。
- 不推荐在页面中嵌入外部第三方资源（除非必要），以降低维护成本。
- 提交更改前请检查页面链接与资源引用是否有效。
- 如有页面废弃，建议移入 `archive/` 目录或直接移除，避免目录膨胀。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 目录下添加或修改 HTML 文件。
3. 通过浏览器打开对应文件即可预览。
4. 若需发布到线上，可将仓库部署至静态托管服务。

## 注意事项

- 本仓库不包含任何动态后端，所有页面均为静态内容。
- 请勿在页面中存储敏感信息，所有内容视为公开。
- 建议使用相对路径引用资源，以保证在不同环境下的兼容性。
